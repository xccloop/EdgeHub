# Kyl-Epoll: LS2K0300 智能车 epoll 可交互串口命令系统

## 项目概述

龙芯 LS2K0300 智能车平台的串口命令交互系统。基于 Linux epoll 机制实现事件驱动架构，
通过 UART1 串口提供实时命令行接口，支持系统状态查询、外设控制等交互功能。

## 项目构想与设计

### 背景

智能车开发过程中，调试和参数调整是高频操作。传统方式需要反复修改代码、
交叉编译、scp 传输、重启程序，整个周期 > 30 秒。更糟糕的是，调试过程中
无法实时观察系统状态（编码器读数、电机 PWM、内存占用等），严重拖慢迭代速度。

### 设计目标

1. **实时交互**：通过串口发送命令，即刻获得响应
2. **非阻塞架构**：epoll 事件驱动，CPU 零空转
3. **可扩展命令**：注册式命令表，新增命令无需改动核心代码
4. **健壮性**：UART 断开自动重连，错误隔离
5. **轻量级**：不依赖 readline/ncurses 等外部库

### 架构设计

```
+----------+     +-----------------+     +-------------------+
| 串口终端  |<--->|   UART1 Driver  |<--->|  Epoll 事件循环    |
| (screen)  |     |  /dev/ttyS1     |     |                   |
+----------+     +-----------------+     +---------+---------+
                                                    |
                                           +--------v--------+
                                           |  Shell 命令解析  |
                                           |  Tokenizer       |
                                           +--------+--------+
                                                    |
                                           +--------v--------+
                                           |  命令处理器       |
                                           |  help/status/... |
                                           +-----------------+
```

### 技术选型

| 组件 | 选型 | 理由 |
|------|------|------|
| I/O 复用 | epoll (Linux) | 专为 Linux 设计，O(1) 事件分发，零拷贝 |
| 触发模式 | 边沿触发 (ET) | 减少 epoll_wait 返回次数，需配合非阻塞 fd |
| 串口 | UART1 /dev/ttyS1 | 板子物理串口，115200 8N1，天然终端 |
| 命令解析 | 自主实现 tokenizer | 80 行代码，无外部依赖 |
| 字符回显 | VT100 控制码 | 支持退格删除，终端体验 |
